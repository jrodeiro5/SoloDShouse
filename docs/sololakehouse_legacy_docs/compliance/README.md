# Compliance Mapping Index

These mappings explain how SoloLakehouse v2.5 supports regulatory-style evidence conversations.

They are not legal advice and do not claim SoloLakehouse makes an institution compliant. The purpose is narrower: show which project components and architecture decisions support auditability, resilience, traceability, and data-record evidence.

Mappings:

- [DORA](dora.md)
- [BaFin BAIT](bafin-bait.md)
- [MiFID II / MiFIR](mifid-ii.md)

Primary regulatory sources:

- DORA: Regulation (EU) 2022/2554, EUR-Lex.
- BAIT: BaFin Circular 10/2017 (BA), updated German version and informational English version.
- MiFID II / MiFIR evidence angle: Regulation (EU) No 600/2014, especially record keeping and transaction-reporting obligations.
