# v2.5 Planning Notes (Iceberg + OpenMetadata + Superset reference)

## Version

- Target version: v2.5 (reference extension between v2 and v3)
- Planning window: 2026-03
- Status: completed

## 1. Goal and constraints

### Goal

- Add Apache Iceberg for the Gold feature table via Trino while keeping Bronze/Silver as Parquet-on-MinIO.
- Add an optional OpenMetadata deployment for metadata discovery, without making catalog migration a v3 prerequisite.
- Add an optional Superset deployment for browser-based SQL / BI exploration over Trino, without changing the core runtime path.

### Non-goals

- Replace Hive Metastore as the system of record.
- Require OpenMetadata for default `make up`.
- Require Superset for default `make up`.
- Migrate Silver to Iceberg in the same release.

### Constraints

- Single-host Docker Compose reference; OpenMetadata and Superset remain profile-gated to keep the default stack lean.

## 2. Outcome

- ADR-013 and ADR-014 accepted; Iceberg, OpenMetadata, and Superset implementation merged on main with compose overlays and documentation updates.
- **Single-track follow-up:** the default `Makefile` always merges `docker-compose.openmetadata.yml` and `docker-compose.superset.yml`, so `make up` starts the full UI stack (no profile gate). Durable local state lives under repository `docker/data/` as bind mounts (not Docker named volumes).
