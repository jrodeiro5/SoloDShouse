## 1. 目标与约束

### 目标

让 SoloLakehouse 在 Compose 单机上具备 **DORA 24h 数据血缘证据响应能力**：任何被监管关注的 Gold 数据集，都能在 60 秒内由一条 `make` 命令产出包含来源、转换、消费者、负责人、时间戳的可签名证据包。

### 不做的事（Non-goals）

| 不做 | 留给 |
|---|---|
| Kubernetes / Helm / Terraform | v3.0 |
| 多环境（dev / staging / prod） | v3.0 |
| Vault / 集中密钥管理 | v3.0 |
| Prometheus / Grafana / 告警接入 | v3.0 |
| 多引擎兼容性 demo（Spark / DuckDB / Flink） | v2.7 |
| 主权报告与退出剧本 | v2.7 |
| ML 模型卡 / 模型 lineage 增强 | v2.8 |
| SLO 指标管道 | v2.9 |
| 推理证据 ID / ComplianceRAG | v3.1+ |
| 模型 serving | 永远 out（ADR-011） |

### 约束

- 单机 Compose 不变
- 不引入新的运行时服务（已有 OpenMetadata + Iceberg + Dagster 三件套就够）
- `make up` 时长不能因此显著增加

## 2. 当前痛点

- OpenMetadata 已部署，但元数据停留在 UI；没有结构化导出
- Iceberg snapshot 自动产生，但与编排上下文无关联
- Dagster run history 在 PG 里，但没有沉淀到不可变审计存储
- 数据集 owner / SLA / quality_class 只在 README/CLAUDE.md 的散文里
- "如果 BaFin 明天来"——目前没有一条命令能给出答案

## 3. 方案

### 核心设计：三源 lineage 融合

```
OpenMetadata (谁负责、SLA、质量等级)
        ┐
Iceberg snapshot (数据版本、提交时间)  ──► LineageRecord JSON ──► WORM audit/
        ┘                                       ▲
Dagster run (编排版本、上游 asset、触发器) ─────┘
```

按 `dataset_id` 关联三源，输出统一 schema，落到 MinIO `audit/` 桶。

### 不引入新组件

OpenMetadata、Iceberg、Dagster 都是 v2.5 已有；v2.6 写的是 **胶水代码 + 数据契约 + CLI**。

## 6. 给下一个版本（v2.7）留下了什么

- 数据契约文件成为 **v2.7 主权报告**的输入（每个数据集的 classification、retention、source_of_truth 直接喂给 `generate-sovereignty-report.py`）
- WORM `audit/` 桶成为 **v2.7 多引擎演示**的旁路验证手段（多引擎读出来的 schema 必须与 audit/lineage 中记录的一致）
- 命名规范成为 **v2.8 ML 绑定**的字段对接基础（`feature_version` 字段引用数据契约 ID）

## 7. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| OpenMetadata REST API 在 1.5.x 上字段不稳定 | lineage_join.py 频繁修补 | 在 join 层引入 adapter 层；关键字段 fail-fast |
| MinIO Object Lock 与 bind-mount 兼容性 | WORM 演示失败 | 提前在 `scripts/prepare-docker-data-dirs.sh` 做能力检测，必要时切换到 docker volume |
| 演练时间超 60 秒 | 监管承诺打折 | 预先把 OpenMetadata 元数据缓存进本地 `audit/cache/`，CLI 优先读缓存 |
| 数据契约文件被当作"另一份 README"维护衰减 | 治理证据失真 | CI 强制：新增 Gold/Silver asset 时，缺契约文件 = build red |

## 8. 技术债与延后项

- 接受：本版没有跨环境治理（只有一个环境）
- 接受：本版没有 RBAC（OpenMetadata 走默认账号）
- 延后到 v2.9：把 `audit/` 桶纳入 SLO（"审计写入失败" 应成为 P1 告警）
- 延后到 v3.1：基于 `audit/lineage/*.json` 训练 LineageNarrator
