## 1. 目标与约束

### 目标

- 让"SLH 不锁定厂商"成为**可演示**，而不是断言：同一张 Iceberg Gold 表，可以被 4 种引擎在 5 分钟内分别读出来
- 让"SLH 100% 欧洲可托管"成为**可签名**：自动生成 `docs/sovereignty-report.md`，供合规直接引用
- 让"客户从 Databricks / Snowflake 迁入 SLH"成为**可执行**：交付迁入 PoC 工具

### 不做的事（Non-goals）

| 不做 | 留给 |
|---|---|
| Kubernetes / Helm | v3.0 |
| 多环境 / 晋升流程 | v3.0 |
| 集中密钥管理 | v3.0 |
| 多用户 / RBAC（Trino LDAP、Superset OAuth） | v3.0 |
| JupyterHub 接入 | v3.0+ |
| TCO 计算器（销售物料） | v3.0+ |

| SLO 与告警 | v2.9 |
| 流式实时管道（Flink streaming demo 仅做演示，不进入主路径） | v3.x+ |

### 约束

- Compose 默认栈不变；多引擎 demo 进入 **独立 compose profile**，不影响 `make up`
- 默认栈启动时间不允许劣化
- Iceberg 表的 schema 与 v2.6 数据契约绑定，多引擎读出的 schema 必须一致

## 2. 当前痛点

- 项目 README 说"开放表格式"，但仓库里没有任何**非 Trino 引擎**的读取 demo
- DACH 客户尽调时常问"如果 SLH 维护中断，我能怎么逃"——目前没有书面剧本
- Iceberg 用 Hive Metastore 作为 catalog，但 REST Catalog 是行业趋势——切换路径未演示
- 主权叙事散落在 README 与 ADR；没有一份"组件—产地—许可证"清单

## 3. 方案

### 三个证据流

```
1. Multi-engine demo            (E8, E15)
   examples/multi-engine/ : Trino / Spark / DuckDB / Flink 读同一张 Iceberg 表
   compose profile: hive-metastore vs iceberg-rest-catalog 切换

2. Sovereignty report            (E6, E7)
   scripts/generate-sovereignty-report.py
   docs/sovereignty-report.md (release 时自动重生成)
   docs/portability-matrix.md

3. Migration-IN tooling          (E14 上半部)
   tools/migrate-from-databricks.py   (Delta → Iceberg 转换)
   docs/exit-playbook.md              (E9)
```

### 关键决策

- **Spark / Flink 不进默认栈**：仅作 `examples/multi-engine/` 下的可选 demo，PR review 不允许把它们引入 `docker/docker-compose.yml`
- **Iceberg REST Catalog 与 Hive Metastore 并存**：通过 compose profile 切换，不替换；让客户自己看哪一种契合他们环境
- **不做 Migration-OUT（SLH → 别处）**：仓库内只演示 IN（迁入 SLH），OUT 由 `exit-playbook.md` 文档化即可——商业上"出"的客户不会主动配合我们做工具

## 6. 给下一个版本（v2.8）留下了什么

- v2.6 的数据契约 + v2.7 的主权报告 = v2.8 模型卡的 **训练数据来源段** 直接复用

## 7. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| Spark / Flink demo 在低配机器跑不动 | 演示失败 | demo 数据集小到 < 10MB；compose profile 默认不启用 |
| Iceberg REST Catalog 与 Hive Metastore 行为差异（如 schema evolution 语义） | 切换叙事打折 | ADR 明确记录差异点；不承诺"完全等价" |
| Migration-IN 工具被当作生产级 ETL 使用 | 客户失望 | 工具自带显眼标语 "PoC only, not for production migration"；不做 schema 自动协商 |
| 主权报告引发法务讨论（如某个 Apache 组件的 contributor 来自非欧洲） | 报告被反复修改 | 报告基于 **维护方总部** 而非 contributor；与法务约定模板 |

## 8. 技术债与延后项

- 接受：本版没有 Migration-OUT 工具
- 接受：Spark / Flink 仅 demo 形态，不做生产部署
- 延后到 v3.0：把多引擎兼容性纳入 release readiness 自动化检查
- 延后到 v3.x：流式管道作为主路径之一（当前仍是批处理优先）
